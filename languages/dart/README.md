# Dart maxspeed target

SofaBuffers `corelib-dart` vs. the official Dart protobuf
([`protoc_plugin`](https://pub.dev/packages/protoc_plugin) generating against the
[`package:protobuf`](https://pub.dev/packages/protobuf) runtime) — the canonical,
by-far-most-used protobuf implementation for Dart, the analogue of prost (Rust)
and protobuf-go (Go). Same message, same values (`schema/STATE.md`), same timed
`encode(decode(wire))` round trip and the same wire gate as every other target.

## AOT, not JIT — on purpose

Both impls are **AOT-compiled to native executables** with `dart compile exe`
and run as binaries; `bench.sh` never uses `dart run`/JIT. This is the fair
comparison to the other compiled ports (C/C++/Rust/Go/Zig), which also run
native — it removes VM warm-up and matches how a deployed Dart binary actually
runs. corelib-dart's own benchmarks note AOT is ≈2× the JIT throughput on these
small-message workloads. The single `dart compile exe` invocation is identical
for both impls, so the row stays apples-to-apples.

## Layout

```
sofab/
  cfg.yaml            emit: project  (sofabgen writes a self-contained Dart pkg)
  bench.dart          our AOT driver — copied into gen/bin/ by setup.sh
  gen/                generated: pubspec.yaml, lib/message.dart, bin/harness.dart
protobuf/
  pubspec.yaml        protobuf + fixnum (hand-written; Renovate-maintained)
  bench.dart          our AOT driver
  gen/message.pb.dart generated by protoc-gen-dart
common/
  bench_common.dart   shared CpuClock + gate SHA-256 (copied next to both drivers)
```

`setup.sh` regenerates both codecs and AOT-compiles them; `bench.sh` runs the two
native binaries and prints the uniform `BENCH` lines. See the repo `README.md`
and `docs/BENCH.md` for the arena contract.

## Notes

- Messages are **hand-filled** from the canonical `STATE.md` values (not parsed
  from `state.json`): Dart's JSON parser loses precision on integer literals
  above 2⁶³−1, so the u64 fields are written from their two's-complement hex bit
  pattern. The cross-language sha256 gate catches any fill drift.
- The `protobuf` runtime version must track the baked `protoc_plugin`
  (`protoc_plugin` 25.x → `protobuf` ^6.0.0); an older major misses `BuilderInfo`
  methods the generated code calls.
